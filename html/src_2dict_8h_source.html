1/* Hash Tables Implementation.
2 *
3 * This file implements in-memory hash tables with insert/del/replace/find/
4 * get-random-element operations. Hash tables will auto-resize if needed
5 * tables of power of two in size are used, collisions are handled by
6 * chaining. See the source code for more information... :)
7 *
8 * Copyright (c) 2006-Present, Redis Ltd.
9 * All rights reserved.
10 *
11 * Licensed under your choice of (a) the Redis Source Available License 2.0
12 * (RSALv2); or (b) the Server Side Public License v1 (SSPLv1); or (c) the
13 * GNU Affero General Public License v3 (AGPLv3).
14 *
15 * Dict usage of pointer tagging
16 * -----------------------------
17 * In the "normal" case (no_value=0), a dict slot contains only a pointer to a
18 * dictEntry, and dictEntry holds untagged pointers to key and value. But when a
19 * dict is used as a set (no_value=1), we optimize by storing direct key pointers
20 * when possible, avoiding dictEntry allocation. This happens when A bucket contains
21 * only one key, or at the tail of a collision chain. Redis dicts uses pointer
22 * tagging, to identify direct key pointers from dictEntry pointers, i.e embedding
23 * metadata in the lowest three bits of pointers. This requires 8-byte alignment,
24 * which zmalloc() guarantees on both 32-bit and 64-bit systems (via jemalloc/tcmalloc,
25 * or standard malloc with explicit PREFIX_SIZE=8).
26 *
27 * Besides of distinguishing direct key pointers from dictEntry pointers, we also
28 * need to distinguish between even and odd key pointers that being stored in the
29 * dict. Therefore, we use the following tagging scheme:
30 * - dictEntry pointer: Points to a dictEntry structure (8-byte aligned). Left intact:
31 * ENTRY_PTR_NORMAL=000
32 * - Odd-address key (keys_are_odd=1): Direct pointer to a
33 * key with odd address (e.g., all SDS strings), Left intact:
34 * ENTRY_PTR_IS_ODD_KEY=XX1
35 * - Even-address key (keys_are_odd=0): Direct pointer to a key with
36 * even address. Since 8-byte alignment yields bits = 000, same as dictEntry,
37 * we tag it by setting bit 1 which results with:
38 * ENTRY_PTR_IS_EVEN_KEY=010.
39 */

56 * ------------------
57 * If dictType.keyFromStoredKey is non-NULL, then dict distinguishes between the
58 * lookup key and the actual stored-key object. In this case, "key" is used to
59 * locate entries, while "storedKey" is the actual element stored in the dict.
60 * If dictType.keyFromStoredKey is NULL, the lookup "key" and the stored-key are the
