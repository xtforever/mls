13/*
14 * script_lua.c unit provides shared functionality between
15 * eval.c and function_lua.c. Functionality provided:
16 *
17 * * Execute Lua code, assuming that the code is located on
18 * the top of the Lua stack. In addition, parsing the execution
19 * result and convert it to the resp and reply ot the client.
20 *
21 * * Run Redis commands from within the Lua code (Including
22 * parsing the reply and create a Lua object out of it).
23 *
24 * * Register Redis API to the Lua interpreter. Only shared
25 * API are registered (API that is only relevant on eval.c
26 * (like debugging) are registered on eval.c).
27 *
28 * Uses script.c for interaction back with Redis.
29 */
