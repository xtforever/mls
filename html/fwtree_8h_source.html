10 * FENWICK TREE (Binary Indexed Tree)
11 * ----------------------------------
12 * A Fenwick tree is a data structure that efficiently supports:
13 * - Point updates: Add/subtract values at specific indices in O(log n) time
14 * - Prefix sum queries: Calculate cumulative sums from index 0 to any index in O(log n) time
15 * - Range queries: Calculate sums over any range [i,j] in O(log n) time
16 * - Space complexity: O(n)
17 *
18 * USAGE IN REDIS
19 * --------------
20 * This implementation is used by KVSTORE and ESTORE to efficiently track:
21 * - Cumulative key counts across dictionary slots (KVSTORE)
22 * - Cumulative item counts across expiration buckets (ESTORE)
23 *
24 * This enables efficient operations like:
25 * - Finding which dictionary/bucket contains the Nth key/item
26 * - Iterating through non-empty dictionaries/buckets
27 * - Load balancing and random key selection
28 *
29 * IMPLEMENTATION NOTES
30 * -------------------
31 * - The tree uses 1-based indexing internally for mathematical convenience
32 * - The public API uses 0-based indexing for consistency with Redis codebase
33 * - Tree size must be a power of 2 (specified as sizeBits where size = 2^sizeBits)
34 * - All operations have O(log n) time complexity where n is the tree size
35 *
36 * REFERENCES
37 * ----------
38 * For more details on Fenwick trees: https://en.wikipedia.org/wiki/Fenwick_tree
39 */
