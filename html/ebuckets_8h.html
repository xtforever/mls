Adds the specified item to the ebucket structure based on expiration time. If the ebucket is a list or empty, it attempts to add the item to the list. Otherwise, it adds the item to rax. If the list reaches its maximum size, it is converted to rax. The ebuckets handler may be updated accordingly.

Parameters
eb- Pointer to the ebuckets handler, which may get updated
type- Pointer to the EbucketsType structure defining the type of ebucket.
item- The eItem to be added to the ebucket.
expireTime- The expiration time of the item.
Returns
0 (C_OK) if the item was successfully added; Otherwise, return -1 (C_ERR) on failure.

◆ ebGetMaxExpireTime()

uint64_t ebGetMaxExpireTime ( ebuckets  eb,
EbucketsType *  type,
int  accurate 
)

Retrieves the expiration time of the item with the latest expiration

However, precision loss (EB_BUCKET_KEY_PRECISION) in rax tree buckets may result in slight inaccuracies, up to a variation of 1<<EB_BUCKET_KEY_PRECISION msec.

Parameters
eb- The ebucket to be checked.
type- Pointer to the EbucketsType structure defining the type of ebucket.
accurate- If 1, then the function will return accurate result. Otherwise, it might return the upper limit with slight inaccuracy of 1<<EB_BUCKET_KEY_PRECISION msec.

This special case is relevant only when the last bucket is of type extended-segment. In this case, we might don't want to traverse the entire bucket to find the accurate expiration time since there might be unbounded number of items in the extended-segment. If EB_BUCKET_KEY_PRECISION is 0, then the function will return accurate result anyway.

Returns
The expiration time of the item with the latest expiration time in the ebucket. If empty, return EB_EXPIRE_TIME_INVALID.

int ebRemove ( ebuckets *  eb,
EbucketsType *  type,
eItem  item 
)

Removes the specified item from the given ebucket, updating the ebuckets handler accordingly. The function is optimized to remove items locally from segments without traversing rax tree or stepping long extended-segments. Therefore, it is assumed that the item is present in the bucket without verification.

Parameters
eb- Pointer to the ebuckets handler, which may get updated if the removal affects the structure.
type- Pointer to the EbucketsType structure defining the type of ebucket.
item- The eItem to be removed from the ebucket.
Returns
1 if the item was successfully removed; otherwise, return 0.
