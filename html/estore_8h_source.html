9 * estore.h -- Expiration Store implementation
10 *
11 * ESTORE (Expiration Store)
12 * =========================
13 *
14 * Index-based expiration store implementation. Similar to kvstore, but built
15 * on top of ebuckets instead of dict. Items stored in estore must embed an
16 * ExpireMeta, enabling efficient active-expiration.
17 *
18 * Estore is currently used to manage "subexpiry" only for hash objects with
19 * field-level expiration (HFE). Each hash with HFE is registered in estore
20 * with the earliest expiration time among its fields.
21 *
22 * USAGE IN REDIS
23 * ==============
24 * This implementation is used to efficiently track hash objects that have
25 * field-level expiration (HFE):
26 * - Each hash with HFE is registered with its earliest field expiration time
27 * - Enables efficient active expiration of hash fields
28 * - Uses Fenwick tree for efficient iteration through non-empty buckets
29 * - Supports cluster mode with per-slot buckets
30 *
31 * IMPLEMENTATION NOTES
32 * ====================
33 * - Built on top of ebuckets data structure for expiration management
34 * - Uses Fenwick tree to track cumulative item counts across buckets
35 * - Supports both single bucket (standalone) and multiple buckets (cluster) modes
36 * - All operations have O(log n) time complexity for bucket selection
37 *
38 * STRUCTURE
39 * =========
40 * - ebArray: Array of ebuckets (one per slot in cluster mode, or just one)
41 * - buckets_sizes: Fenwick tree tracking cumulative counts for efficient iteration
42 * - bucket_type: EbucketsType defining callbacks for the stored items
43 */
